Copia el valor y prioridad de cada elemento en arreglos externos.

Copia valor/prioridad de cada elemento.

Recorre la cola de frente a fondo. Por cada nodo, copia su valor en valores[i] y su prioridad en prioridades[i]. Se copian como máximo capacidad elementos. La cola no se modifica.

Parámetros
[in]colaPuntero constante a la ColaPrioridad de origen.
[out]valoresArreglo donde se almacenarán los valores copiados.
[out]prioridadesArreglo donde se almacenarán las prioridades.
[in]capacidadNúmero máximo de elementos a copiar.
Devuelve
Número de elementos efectivamente copiados. Retorna 0 si algún parámetro es inválido.
Precondición
valores y prioridades deben apuntar a bloques con espacio para al menos capacidad enteros cada uno.
Nota
Complejidad temporal: O(min(n, capacidad)).

◆ cp_desencolar()

bool cp_desencolar ( ColaPrioridad * cola,
int * valor,
int * prioridad )

Desencola el elemento de mayor prioridad efectiva.

Recorre toda la cola buscando el nodo con el valor de prioridad más bajo (número más pequeño). En caso de empate extrae el que fue insertado primero (el que aparece antes en la lista). Una vez encontrado, lo desenlaza actualizando delante o atras según corresponda, escribe sus datos en los punteros valor y prioridad y libera su memoria.

Parámetros
[in,out]colaPuntero a la ColaPrioridad de origen.
[out]valorPuntero donde se escribe el valor del nodo extraído.
[out]prioridadPuntero donde se escribe la prioridad extraída.
Devuelve
true si se extrajo un elemento correctamente.
false si cola, valor o prioridad son NULL, o la cola está vacía.
Precondición
La cola debe contener al menos un elemento.
Postcondición
El tamaño de la cola disminuye en 1.
Nota
Complejidad temporal: O(n), donde n es el número de elementos.

bool cp_encolar ( ColaPrioridad * cola,
int valor,
int prioridad )

Encola un elemento con su valor y prioridad asociada.

Encola un valor con prioridad asociada.

Reserva dinámicamente un nuevo CPNodo, almacena valor y prioridad en él y lo enlaza al extremo atras de la cola. Si la cola estaba vacía, tanto delante como atras apuntarán al nuevo nodo. El orden de extracción depende de la prioridad, no del orden de inserción.

Parámetros
[in,out]colaPuntero a la ColaPrioridad destino.
[in]valorDato entero a almacenar.
[in]prioridadNúmero de prioridad (menor valor = mayor prioridad).
Devuelve
true si el nodo fue creado e insertado correctamente.
false si cola es NULL o malloc() falla.
Precondición
La cola debe haber sido inicializada con cp_inicializar().
Postcondición
El tamaño de la cola aumenta en 1.
Nota
Complejidad temporal: O(1).

◆ cp_formatear()

void cp_formatear ( const ColaPrioridad * cola,
char * destino,
size_t capacidad )

Genera una representación textual de la cola de prioridad.

Genera una representacion textual de la cola.

Escribe en destino una cadena con el formato:

frente -> v1(p=p1) | v2(p=p2) | ... | vN(p=pN)

donde vK es el valor y pK la prioridad del k-ésimo elemento en orden de inserción. Si la cola está vacía escribe "Cola de prioridad vacia". La cadena siempre queda terminada en '\0' si capacidad >= 1.

Parámetros
[in]colaPuntero constante a la ColaPrioridad.
[out]destinoBuffer donde se escribirá la cadena resultante.
[in]capacidadTamaño en bytes del buffer destino.
Precondición
destino debe apuntar a un buffer de al menos capacidad bytes.
Postcondición
destino contiene una cadena terminada en '\0'.
Nota
Si el contenido supera capacidad se trunca de forma segura.
Si destino es NULL o capacidad es 0, no hace nada.
Complejidad temporal: O(n).
